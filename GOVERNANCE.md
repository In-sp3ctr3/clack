# Governance

Clack currently uses a maintainer-led model.

## Maintainer

- Jadan Jones, `@In-sp3ctr3`

## Decision Making

The maintainer makes final calls on product direction, releases, project standards, and pull request merges. Community feedback is welcome, especially around UX, privacy, packaging, and macOS edge cases.

## Contribution Model

Contributors can help through issues, discussions, pull requests, testing, documentation, and design feedback. Larger changes should start as an issue or discussion before implementation.

## Releases

Releases are cut by maintainers from the protected `main` branch. Release notes should summarize user-facing changes, fixes, and known limitations.

## Changes to Governance

This file can change as the project grows. Material governance changes should happen through a pull request.
